# KY-MVC
MVC
MODEL adalah bagian dalam konsep MVC yang bertugas mengelola data dan logika aplikasi. Di dalam Model terdapat proses penyimpanan, pengambilan, dan pengolahan data, biasanya yang berhubungan dengan database. Model tidak berinteraksi langsung dengan pengguna dan tidak mengatur tampilan, karena fokus utamanya adalah memastikan data yang digunakan aplikasi sudah benar dan sesuai aturan yang ditentukan. VIEW adalah bagian yang bertugas menampilkan data kepada pengguna. View menyajikan informasi dalam bentuk yang bisa dilihat, seperti teks, tabel, atau halaman aplikasi. View tidak melakukan perhitungan atau pengolahan data, melainkan hanya menerima data dari Controller lalu menampilkannya agar mudah dipahami oleh pengguna. CONTROLLER adalah bagian yang berfungsi sebagai penghubung antara Model dan View. Controller menerima perintah atau input dari pengguna, kemudian memprosesnya dengan memanggil Model untuk mengambil atau mengolah data. Setelah data diperoleh, Controller mengirimkan hasilnya ke View untuk ditampilkan, sehingga alur kerja aplikasi dapat berjalan dengan teratur dan terstruktur..
